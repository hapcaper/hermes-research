**核心优势：不需要熟人网络**
- 开发者工具靠技术内容传播（GitHub、掘金、知乎）

- 李梓浩的Java后端技术栈完全匹配

**三大规律（已验证）：**
1. **开发者工具最适合独立开发者** —— PLG模式，自带传播属性
2. **月入3K-30K是独立开发者最现实的"舒适区"** —— 单人或2-3人小团队
3. **出海+美元定价能显著提升收入质量** —— $1000/月 ≈ 7200元，技术壁垒不需很高

**真实案例参考（V2EX 2025年）：**
- HexHub：开发者工作台（SSH、数据库、Docker、AI Agent），有人在做

**李梓浩能做的工具类型（按难度排序）：**

**Top 6 合同风险点（按纠纷频率排序）**：
1. 🔴 工程款支付条款（40-50%纠纷）

### 2026-05-13 第二十期：AI数据清洗工具 — 技术验证完成

#### Spike 008 技术验证结果

**问题**：给定一个乱糟糟的CSV数据，当用AI数据清洗工具处理时，能否自动检测并修复常见数据质量问题？

**验证方法**：用纯Python（无pandas依赖）实现了完整的数据清洗管道，测试6条真实数据（含空值、格式不一致、重复行等）

**测试数据**：
```

C001,张三,13812345678,2025-01-15,1500,zhang@test.com
C002,李四 ,139-5678-1234,01/20/2025,2000, li@test.com
C003,,+86 13712345678,2025/01/30,1800,zhao@test.com
C004,赵六,13812345678,20250205,1600,sun@test.com
C005,孙七,13612345678,2025-02-10,2100,sun@test.com
C001,张三,13812345678,2025-01-15,1500,zhang@test.com  ← 重复行
```

**检测到的6个问题**：
| # | 问题类型 | 列 | 严重度 | 状态 |
|---|---------|-----|--------|------|
| 1 | EMPTY_CELLS | name | HIGH | ✅ 修复成功 |
| 2 | INCONSISTENT_FORMAT | phone | MEDIUM | ✅ 标准化成功 |
| 3 | INCONSISTENT_DATE | join_date | HIGH | ✅ 标准化成功 |
| 4 | DUPLICATES | ALL | HIGH | ✅ 删除成功 |
| 5 | WHITESPACE | name | LOW | ✅ 修复成功 |
| 6 | WHITESPACE | email | LOW | ✅ 修复成功 |

**清洗后**：6 rows → 5 rows，Quality Score: 86.1/100

**技术亮点**：
- 纯Python实现，无pandas依赖（pip install都能跑）

**谁在做这个生意？**

1. 部署文档太完善，5年经验的Java后端看2小时文档就会
2. 客户可以自己看文档部署，或让公司IT做
3. 没有不可替代性 —— 任何会Docker的后端都能做
4. 没有复利效应 —— 每单都是新单，不存在"睡后收入"

##### 发现5：维护费（500-2000元/月）是幻觉

**为什么维护费很难收？**
- 部署完成后，除非知识库内容更新，否则没有维护需求
- 知识库内容更新通常是客户自己操作（上传文档）
- LLM API出问题，客户自己找云服务商解决
- **一次性服务，没有持续价值**

**什么时候维护费真实存在？**
- 客户每月有新的知识库更新需求
- 客户需要定期优化知识库检索效果
- 客户需要新增AI应用场景
- 但以上都需要客户主动找你才行

#### 方向二：SQL 审核工具（SQL Review for CI/CD）

**一句话**：在代码提交前自动审核 SQL 质量，检测性能风险（如 SELECT *、缺少索引、全表扫描）。

**为什么这个方向可行：**

1. **真实痛点**：线上 SQL 慢查询是 DBA 最头疼的问题，通常在上线后才发现
2. **技术方案**：基于 TiDB / Slow Query Log 分析 + SQL 静态规则
3. **差异化空间**：不做"SQL编辑器"（有很多），专门做"SQL审核+拦截"
4. **定价参考**：199-499元/月（团队版）

**MVP 范围：**
- 输入：SQL 语句或 MySQL 连接信息
- 核心功能：识别性能风险（缺少索引、SELECT *、子查询优化建议）
- 输出：风险报告 + 优化建议

**竞品情况：**
- Yearning：开源 SQL 审核平台，面向 DBA 团队
- AI2SQL：AI 生成 SQL，非审核方向
- 仙共同学会 SQL 审核：内部工具，非商业产品

**关键优势：**
- 可以跟 CI/CD 集成（Git hook），在代码提交阶段拦截问题 SQL
- 可以针对中国特色 SQL 场景（MySQL 5.7/8.0 兼容）

**预估收益：** 月 3K-20K（20-50个团队 × 199-499元）
