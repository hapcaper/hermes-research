### 2025-05-15 第九期：建筑合同方向彻底放弃 + 开发者工具新方向确立

#### ⚠️⚠️⚠️ 致命发现：所有竞品从未真正上线运营

**DNS调查彻底揭示真相：**

| 竞品 | 域名 | DNS解析结果 | 真相 |
|------|------|-------------|------|
| orm.cn | www.orm.cn | 198.18.2.156 | RFC 2544基准测试IP，非真实服务器 |
| otherm.cn | www.otherm.cn | 198.18.2.107 | 同上 |
| functek.cn | www.functek.cn | 198.18.2.145 | 同上 |

**198.18.x.x是什么？**
- IANA保留地址段（RFC 2544）
- 专门用于网络基准测试和性能评估
- **任何生产环境都不会使用这个IP段**
- 这意味着这三个竞品域名从未配置过真实的生产服务器

**这彻底推翻了什么？**
- ❌ "otherm.cn证明市场存在"——他们从未真正运营过
- ❌ "orm.cn敢收99-999元/月"——他们可能从未收到过任何钱
- ❌ "Functek是真实竞品"——他们的网站从未真正上线

**结论：建筑合同风险识别市场是一个"理论上的市场"，没有任何真实运营的竞品可以验证。**

- 部署文档齐全，步骤标准化

- **部署的技术壁垒很低，低到李梓浩能做的任何公司IT也能做**

##### 发现4：市场存在，但壁垒极低

**谁在做这个生意？**
- 云服务商（阿里云、腾讯云、AWS）—— 卖AI解决方案，捆绑卖云资源
- 软件外包公司 —— 打包卖"AI数字化转型"
- 独立开发者/工作室 —— 赚快钱，一单3000-8000元

**为什么壁垒极低？**
1. 部署文档太完善，5年经验的Java后端看2小时文档就会
2. 客户可以自己看文档部署，或让公司IT做
3. 没有不可替代性 —— 任何会Docker的后端都能做
4. 没有复利效应 —— 每单都是新单，不存在"睡后收入"

##### 发现5：维护费（500-2000元/月）是幻觉

**为什么维护费很难收？**
- 部署完成后，除非知识库内容更新，否则没有维护需求
- 知识库内容更新通常是客户自己操作（上传文档）
- LLM API出问题，客户自己找云服务商解决
- **一次性服务，没有持续价值**

**什么时候维护费真实存在？**
- 客户每月有新的知识库更新需求
- 客户需要定期优化知识库检索效果
- 客户需要新增AI应用场景
- 但以上都需要客户主动找你才行

#### 方向一：API 变更日志追踪器（API Change Log Tracker）

**一句话**：自动追踪团队 API 文档变化，生成变更记录，供前端/客户端团队查阅。

**为什么这个方向可行：**

3. **技术方案**：接入 Swagger/OpenAPI，自动比较版本差异，生成 changelog
4. **定价参考**：99-299元/月（个人/小团队）

**MVP 范围：**
- 输入：Swagger/OpenAPI JSON URL
- 核心功能：比较两个版本的 API diff，输出变更说明
- 附加：生成 changelog 页面，支持分享链接

**竞品情况：**
- SwaggerHub 有版本管理但不是这个定位
- Apifox 有变更记录但不是核心功能
- 未发现专门做"API变更追踪+通知"的独立产品

**风险：**
- 需要持续接入用户项目（集成成本）
- 可能的替代：直接用 Apifox 的版本管理功能

#### 方向二：SQL 审核工具（SQL Review for CI/CD）

**一句话**：在代码提交前自动审核 SQL 质量，检测性能风险（如 SELECT *、缺少索引、全表扫描）。

**为什么这个方向可行：**

1. **真实痛点**：线上 SQL 慢查询是 DBA 最头疼的问题，通常在上线后才发现
2. **技术方案**：基于 TiDB / Slow Query Log 分析 + SQL 静态规则
3. **差异化空间**：不做"SQL编辑器"（有很多），专门做"SQL审核+拦截"
